# servo_control_with_feedback_arduino

This repository contains `servo_control_with_feedback_arduino`, an Arduino sketch for **generic servo control with motion shaping** and **real position feedback** read from the servo’s internal potentiometer.

The sketch generates a smooth commanded reference (software motion profile), drives the servo via PWM, reads the internal potentiometer signal through an ADC input, and prints aligned telemetry over USB Serial.

## What the sketch does

The commanded reference is produced by a trapezoidal motion profile defined by:

- Target angle (degrees)
- Maximum velocity limit (**V%**, 1–100%)
- Maximum acceleration limit (**A%**, 1–100%)

At the same time, the sketch measures the actual servo position using the internal potentiometer (connected to `A4`) and computes:

`Err = Cmd − Meas`

Where:

- `Cmd` is the instantaneous reference generated by the motion profile (not just the final target),
- `Meas` is the measured angle from the internal potentiometer.

## Hardware setup

An Arduino Uno (or compatible 5 V board), one standard RC servo, and three user potentiometers are required.

Connections:

- Servo signal → `D9` (configurable as `SERVO_PIN`)
- Servo power → external supply recommended
- Servo ground → supply ground
- Arduino ground → must be connected to the same ground as the servo supply

User potentiometers (0–5 V dividers):

- Target angle → `A0`
- Vmax% (speed limit) → `A1`
- Amax% (accel limit) → `A2`

Servo internal feedback (potentiometer wiper):

- Wiper → `A4`
- Recommended analog filter at Arduino input:
  - 1 kΩ series resistor from wiper to `A4`
  - 4.7 µF capacitor from `A4` to GND

The RC filter reduces noise and interference from the servo electronics.

High-current servos must not be powered from the Arduino 5 V rail.

## Control principle

At a fixed interval (`LOOP_INTERVAL_MS`), the sketch:

1. Reads target angle from `A0` and maps it to 0–180°.
2. Reads Vmax% (`A1`) and Amax% (`A2`) and maps them to integer percentages (1–100%).
3. Converts percentages into physical limits used by the profile:
   - `vmaxDegps = SERVO_MAX_SPEED_DEGPS * (V% / 100)`
   - `accelDegps2 = ACCEL_MAX_DEGPS2 * (A% / 100)`
4. Updates the internal commanded reference (`cmdDeg`) using a trapezoidal motion profile.
5. Converts `cmdDeg` to calibrated PWM microseconds and sends it to the servo.
6. Reads the feedback ADC (`A4`), converts it to degrees, and computes error `Err = Cmd − Meas`.
7. Prints fixed-width telemetry (aligned columns).

The trapezoidal profile uses:

- `v = v + a·dt` as the acceleration ramp
- Velocity limiting by `vmaxDegps`
- Stopping constraint `v_stop = sqrt(2·a·distance)` to guarantee deceleration to the target

This produces smooth acceleration and deceleration of the commanded reference.

## Startup behavior

On power-up, the sketch reads the servo feedback potentiometer and initializes the commanded reference (`cmdDeg`) to the measured angle. This prevents a startup jump and begins from the actual physical position.

## Servo speed parameter (SERVO_MAX_SPEED_DEGPS)

The sketch uses one physical characterization parameter:

`SERVO_MAX_SPEED_DEGPS` (deg/s)

It represents the **maximum real angular speed** of the servo for the current servo model, supply voltage, and conditions.

Example conversions from datasheet-style specifications:

- HS-805BB @6V  
  `0.14 s / 60°  ->  60 / 0.14  = 428.6 deg/s`

- Futaba S3003 @4.8V  
  `0.23 s / 60°  ->  60 / 0.23  = 261.0 deg/s`  (**default used in the sketch**)

If you change the servo model, supply voltage, or conditions that affect speed, update `SERVO_MAX_SPEED_DEGPS`.

## Servo calibration

PWM calibration (microseconds) is defined by:

- `SERVO_CENTER_US` (typically ~1500 µs)
- `SERVO_HALFSPAN_US` (span around center)

These parameters define `PWM_MIN_US` and `PWM_MAX_US` and must be chosen conservatively to avoid driving into mechanical hard stops.

Feedback ADC calibration uses two measured points:

- `FB_ADC_AT_0_DEG`
- `FB_ADC_AT_180_DEG`

These values are used to map ADC readings into degrees via a two-point linear calibration. They must be updated if another servo is used or if the feedback wiring differs.

## Serial output

At each control update (115200 baud over USB), the sketch prints aligned telemetry with columns:

`Target | Cmd | Meas | Err | V | V% | A% | PWM | ADC`

Meaning:

- `Target` : target angle from the target potentiometer (deg)
- `Cmd`    : commanded reference angle from the motion profile (deg)
- `Meas`   : measured real angle from servo internal potentiometer (deg)
- `Err`    : tracking error `Cmd − Meas` (deg)
- `V`      : instantaneous profile velocity (deg/s)
- `V%`     : speed limit knob (1–100%) of `SERVO_MAX_SPEED_DEGPS`
- `A%`     : accel limit knob (1–100%) of `ACCEL_MAX_DEGPS2`
- `PWM`    : PWM command sent to the servo (µs)
- `ADC`    : raw averaged ADC reading from the feedback pin (0–1023)

The fixed-width formatting keeps values aligned in a Serial Monitor and simplifies copy/paste logging.
